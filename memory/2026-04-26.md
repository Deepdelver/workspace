# 2026-04-26 - Daily Log

## Morning - Kubernetes GPU Fix (06:53 AM)

### Probleem
- Cron job `hourly-git-sync` gedraaid
- Ontdekte dat k8s-stack repo niet synced was met cluster
- Ollama deployment had GPU issues: pods bleven Pending met "Insufficient nvidia.com/gpu"
- k8s-stack repo had `nvidia.com/gpu: null` vs cluster had `nvidia.com/gpu: "1"`

### Root Cause Analysis
1. NVIDIA device plugin was niet actief in de cluster
2. Node rapporteerde `nvidia.com/gpu: 0` in Capacity/Allocatable
3. Host heeft wel NVIDIA devices (/dev/nvidia0 etc.) en drivers werken (nvidia-smi werkt)
4. NVIDIA GPU operator was nog niet geïnstalleerd via ArgoCD

### Oplossing
1. Gevonden dat `/home/frank/k8s-stack/base/argocd/applications/nvidia-gpu-operator.yaml` bestond maar niet gesynced was
2. ArgoCD Application handmatig aangemaakt: `kubectl apply -f [file]`
3. ArgoCD synced automatisch (selfHeal: true)
4. NVIDIA GPU operator geïnstalleerd in `gpu-operator-resources` namespace
5. Na installatie: node rapporteert `nvidia.com/gpu: 1`
6. Ollama pod startte succesvol met GPU support

### Deployment Strategy Fix
- Patch toegepast op ollama deployment: `maxSurge: 0, maxUnavailable: 1`
- Dit voorkomt dat er 2 pods tegelijk starten bij updates (probleem bij GPU scheduling)


## Lessons Learned
- Bij GPU workloads: altijd checken of NVIDIA device plugin daadwerkelijk draait
- `nvidia-smi` op host werkt niet garantie dat Kubernetes GPU ziet
- Check altijd: `kubectl describe node | grep nvidia.com/gpu`
- ArgoCD Applications kunnen "Missing" zijn als ze niet gesynced zijn
- GPU scheduling vereist `maxSurge: 0` bij rolling updates
