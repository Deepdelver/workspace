## Sessie 18 april 2026

### Zigbee2MQTT WC/Gang lampen offline
- **Probleem**: `light.wc_light` en `light.gang` tonen "unavailable"
- **Diagnose**: `last_seen` in Zigbee2MQTT state.json toonde >33 dagen geleden - lampen waren fysiek offline
- **Oplossing**: WC lamp hersteld door `switch.wc_switch_switch_1` aan te zetten via HA API
- **Gang lamp**: Moet fysiek wandschakelaar omgezet worden
- **Leerpunt**: Bij "unavailable" Zigbee lampen: altijd eerst wandschakelaars + `last_seen` checken

### Commands used
- `curl -X POST ... switch/turn_on` voor WC switch
- `ssh root@192.168.1.251 cat /config/zigbee2mqtt/state.json` voor device status
- `curl -s -H "Authorization: Bearer ..." https://.../api/states/light.wc_light` voor entity check

## Avond 18 april 2026 - Home Assistant Automatie Problemen

### Probleem 1: Automations niet zichtbaar in frontend
- **Symptoom**: HA toonde maar 3 automations, API toonde 76
- **Oorzaak**: configuration.yaml gebruikte `!include automations.yaml` i.p.v. `!include_dir_merge_list automation`
- **Fix**: Veranderd naar `!include_dir_merge_list automation` en backup gekopieerd naar /config/automation/

### Probleem 2: Keuken timeout te kort
- **Symptoom**: Keuken licht ging na 10 seconden uit
- **Fix**: `keuken_beweging_verlengd` timeout veranderd van 10 naar 15 minuten

### Probleem 3: Hal automatie werkte niet
- **Symptoom**: Automatie werd getriggerd maar switch ging niet aan
- **Oorzaak**: Device-based automatie met verouderde device IDs
- **Fix probeersel**: Omgezet naar entity-based (later kapotgemaakt door YAML fout)

### Probleem 4: YAML syntax error → Herstelmodus
- **Oorzaak**: Foute YAML structuur in mijn bewerkingen
- **Fix**: Originele backup hersteld
- **Leerpunt**: Voorzichtig met direct YAML bewerken in HA config

### Probleem 5: Gang fysieke schakelaar
- **Symptoom**: Gang switch en light zijn "unavailable"
- **Oorzaak**: Fysieke wandschakelaar werkt niet / is stuk
- **Oplossing**: Morgen fysiek checken

### Probleem 6: Buddy Dashboard Alarm Card
- **Symptoom:** alarm-control-panel geeft parsing error in buddy-yaml dashboard
- **Entity:** `alarm_control_panel.homebase` (werkt - status: armed_away)
- **Oorzaak:** Home Assistant bug met YAML-mode dashboards
- **Workaround:** Card tijdelijk uitgeschakeld (#- type: alarm-control-panel)
- **Oplossing:** Schakel over naar UI-mode OF gebruik custom alarm-card via HACS
