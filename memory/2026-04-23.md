## LiteLLM Router Deployment - Kubernetes Issues & Fixes

### Problem Summary
- **Readiness Probe Failure**: The Litellm-router pod was stuck with `READY=False` because the readiness probe (`tcpSocket:4000`) failed with `"connect: connection refused"`.
- **Root Cause**: The Litellm router was not properly binding to port 4000, likely due to model initialization failures or missing API keys in `litellm-secrets`.

### Key Findings
- **19 Models Configured**: The router includes 3 Gemini variants (with 3 API keys), 3 Groq variants, and other models like GPT-4o mini, Claude, DeepSeek, and ZeN.
- **API Keys**: All Gemini API keys (`GEMINI_API_KEY`, `GEMINI_API_KEY_2`, `GEMINI_API_KEY_3`) were added to `litellm-secrets` but may not have been properly referenced in `litellm-config`.
- **Deployment Stuck**: Kubernetes was unable to terminate old replicas, causing a rollout timeout.

### Fixes Applied
1. **Forced Deployment Restart**: Patched the deployment to remove the `restartedAt` annotation and restarted it to bypass the timeout.
2. **ConfigMap Validation**: Verified that the `litellm-config` ConfigMap includes all model entries with correct API key references.
3. **Model List Management**: Planned to move the model list directly into `litellm-config` for explicit control instead of relying on `litellm-secrets`.

### Commands Used
- `kubectl patch deployment litellm-router --namespace=litellm-router --type='json' -p='[{"op":"remove","path":"/metadata/annotations/kubectl.kubernetes.io/restartedAt"}]'`

- `kubectl get secret litellm-secrets --namespace=litellm-router -o yaml`

## Lessons Learned
- **Readiness Probe**: Always ensure the application is properly binding to the specified port before Kubernetes considers it ready.
- **API Key Management**: Store API keys in secrets but ensure they are correctly referenced in the ConfigMap.
- **Deployment Rollouts**: Kubernetes rollouts can get stuck if old replicas don't terminate; force restarts may be necessary.
- **Model List Validation**: Validate the model list in the ConfigMap to ensure all models are properly configured and referenced.
