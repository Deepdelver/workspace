## Extended Debugging Session - Evening (19:00-22:35)

### Duration: 6.5 hours (too long!)

### Issues Tackled:
1. **litellm-router** - Prisma migration takes 2+ minutes, original readiness probe initialDelay 20s too short. Also env var DATABASE_URL conflict (sqlite vs postgresql). Eventually fixed by:
   - Removing DATABASE_URL from deployment env
   - Patching readiness probe initialDelaySeconds to 180
   - Directly patching deployment with kubectl (bypassing ArgoCD sync lock)
   - Result: pod 1/1 Ready at 22:35.

2. **ArgoCD sync lock** - crossplane-helm app stuck in deletion for 2+ hours, blocking all other syncs. Fixed by:
   - Removing finalizers via patch
   - Deleting ApplicationSet temporarily
   - Force deletion

3. **openclaw-operator** - Added missing CRD (openclawselfconfigs.yaml) to deployment.

4. **sealed-secrets** - Fixed Server-Side Apply conflicts.

### Lessons Learned (Hard Way):
- **Direct action**: Don't spend 6.5 hours debugging. Set a 30-minute limit per issue, then escalate or change approach.
- **Probe timing**: For apps with slow startup (like Prisma migration), set initialDelaySeconds appropriately (180s).
- **Env var overrides**: Environment variables can override ConfigMap values, causing hidden conflicts.
- **ArgoCD locks**: One stuck app can block entire sync. Kill quickly with `kubectl patch` to remove finalizers.
- **Pragmatism**: Sometimes it's faster to delete and recreate resources than to debug.

### User Feedback:

Adopt a more direct, action-oriented approach. No more marathon debugging sessions.
